Documentation

Welcome to the AnonyChat technical documentation. This guide will walk you through setting up the application from scratch on various environments.

AnonyChat is a privacy-focused messaging application built with Laravel and real-time WebSockets.

Prerequisites

Before you begin, ensure your server meets the following requirements:

Server Software

  • PHP 8.2 or higher
  • MySQL 8.0+ or MariaDB
  • Composer 2.x
  • Node.js & NPM (for assets)

PHP Extensions

BCMath, Ctype, Fileinfo, Mbstring, OpenSSL, PDO, Tokenizer, XML, Curl, Dom.

Using the Installer Wizard

We've included a standalone installation wizard to automate the heavy lifting.

New: The wizard now supports setup via IP address if you haven't pointed a domain yet.
  1. Upload the project to your web server (see Zip Deployment below).
  2. Navigate to your domain or IP: http://your-ip-address/install/
  3. Follow the step-by-step guide to:
    • Check server compatibility.
    • Configure your database connection.
    • Setup WebSocket variables (Use the IP address if no domain exists).
    • Run migrations and generate application keys.

Deploying from a Zip Archive

If you've received AnonyChat as a .zip or .tar.gz archive, follow these steps to prepare the files:

1. Upload & Extract

Upload the archive to your server's web root (e.g., /var/www/html on Linux or C:\inetpub\wwwroot on Windows).

# On Linux via SSH:
unzip anonychat.zip -d /var/www/anonychat

2. Set Permissions

The web server needs write access to specific folders to store logs and cache.

Linux (Nginx/Apache) 
sudo chown -R www-data:www-data /var/www/anonychat
sudo chmod -R 775 /var/www/anonychat/storage
sudo chmod -R 775 /var/www/anonychat/bootstrap/cache
Windows (IIS)

Right-click storage and bootstrap/cache, go to Properties > Security, and add Full Control for the IIS_IUSRS user.

Manual Dependency Setup

If you prefer to set up the dependencies manually or are developing locally, run these commands in the web/ directory:

# Install PHP dependencies
composer install

# Install Node.js dependencies (use --legacy-peer-deps if conflicts occur)
npm install --legacy-peer-deps

# Build frontend assets
npm run build

# Generate application key
php artisan key:generate

Local Development Server

To quickly launch the installation wizard or the application locally without configuring Nginx/Apache/IIS:

# Start PHP's built-in server
php -S localhost:8000

Then navigate to http://localhost:8000/install/ in your browser.

Accessing via IP Address (No Domain)

You can run AnonyChat perfectly fine using just your server's Public IP address. This is great for private setups or initial testing.

1. Application URL

In the Installation Wizard, when asked for the App URL, enter your IP address including the protocol:

APP_URL=http://68.183.191.161

2. WebSocket Configuration

For Real-time messaging to work on an IP-only server, ensure your REVERB_HOST is set to the same IP address.

Protocol Note: If you are accessing via IP without an SSL certificate, you must use ws:// (Port 8080) instead of wss:// (Port 443). Browsers generally allow insecure WebSockets ONLY if the main site is also on HTTP.

3. Firewall Setup

Ensure your server allows traffic on necessary ports:

  • Port 80/443: Standard web access.
  • Port 8080: Default port for Reverb/WebSockets.

Linux Setup

Ubuntu / Debian / CentOS

For Linux environments, we recommend using Nginx as a reverse proxy.

Nginx Configuration

server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/anonychat/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }
}

Apache Configuration

Ensure mod_rewrite is enabled. The public/.htaccess file included with AnonyChat handles the routing automatically.

sudo a2enmod rewrite
sudo systemctl restart apache2

Windows Setup (IIS)

Windows Server / Windows 10/11

Running Laravel on IIS requires the URL Rewrite Module.

  1. Install PHP for Windows and link it to IIS via FastCGI.
  2. Download and install the URL Rewrite Module.
  3. Create a new Site in IIS pointing to the public/ folder.
  4. IIS will automatically pick up the public/web.config file.
Permission Hint: Give IIS_IUSRS Full Control over the storage and bootstrap/cache directories.

Testing on Virtual Machines

If you wish to test the system on a VM (VirtualBox, VMware, etc.):

  • Linux VM: Use a clean Ubuntu Server 22.04 LTS instance. Use ssh to connect and follow the Linux Setup instructions.
  • Windows VM: Ensure you enable the "Web Server (IIS)" role in Server Manager before starting.
  • Networking: Set your VM network adapter to "Bridged" to access the installer from your host machine via its local IP.

WebSockets (Laravel Reverb)

Laravel Reverb is a first-party WebSocket server for Laravel applications.

WS vs WSS Protocol

WS (ws://)

Standard unencrypted connection. Should ONLY be used for local development or behind a trusted local proxy.

WSS (wss://)

Secure WebSocket. Required if your website is running on HTTPS. Most modern browsers will fail to connect if you try to use WS on an HTTPS site.

Production WebSocket Setup

In production, you should run the Reverb server on a high port (e.g. 8080) and use your web server (Nginx/Apache) to proxy the traffic to it under SSL (Port 443).

# Start the Reverb server
php artisan reverb:start

Post-Installation

Once the installer has finished, perform these final steps:
  • Security: Delete the install/ directory immediately.
  • Background Tasks: Setup a Cron job to run the Laravel Scheduler every minute: 
    * * * * * cd /path-to-your-project && php artisan schedule:run >> /dev/null 2>&1
  • Persistence: Use Supervisor (Linux) or a Windows Service to keep php artisan queue:work and php artisan reverb:start running 24/7.

Mobile Documentation

Welcome to the AnonyChat Mobile development guide. The mobile application is built using Flutter, providing a fast, cross-platform experience for Android and iOS.

AnonyChat Mobile connects seamlessly to the Laravel backend for secure, real-time messaging.

Requirements

To develop or build the AnonyChat Mobile app, you need the following tools:

Flutter SDK

  • Flutter 3.x (Stable)
  • Dart 3.x

Platforms

  • Android: Android Studio, Java JDK 17+, Android SDK
  • iOS: macOS with Xcode 15+ (Required for iOS)

Development Setup

If you received the code as a .zip file, follow these steps to get started:

1. Extract & Open

Extract the mobile/ folder to your project directory. Open the folder in VS Code or Android Studio.

2. Install Dependencies

Run the following command in the terminal inside the mobile/ directory:

flutter pub get

3. Configure Environment

Open lib/services/api_service.dart and set your backend URL:

static const String baseUrl = 'https://chat.orellepos.com/api';

4. Run on Device

Connect your physical device or start an emulator and run:

# Run in Debug Mode
flutter run

Android Build

Windows / Linux / macOS

To create a production-ready APK or App Bundle for Android:

Build APK

flutter build apk --release

The generated file will be at: build/app/outputs/flutter-apk/app-release.apk

Build App Bundle (for Play Store)

flutter build appbundle --release

iOS Build

macOS Only

Building for iOS requires a Mac with Xcode installed.

1. Prepare CocoaPods

cd ios
pod install
cd ..

2. Build IPA

flutter build ipa --release

You can then find the .xcarchive and export it via Xcode's Organizer to create an IPA for distribution.

Backend Integration

The app communicates with the Laravel backend via a RESTful API. Ensure the following are configured correctly:

Authentication: The app uses Laravel Sanctum. Token storage is handled automatically via GetStorage.

Critical Files

  • lib/services/api_service.dart: Base API configuration.
  • lib/controllers/auth_controller.dart: Handles Login/Register/Logout.
  • lib/controllers/chat_controller.dart: Handles message sending and read receipts.

WebSockets & Real-time

Real-time updates are powered by Laravel Reverb and the laravel_echo package.

Echo Configuration

Edit lib/services/api_service.dart (or the dedicated Echo service) to match your Reverb settings:

// Reverb Connection Config
'host': 'chat.orellepos.com',
'port': 443,
'encrypted': true,
'wsPort': 443,
'wssPort': 443,
Read Status Logic: Read receipts are sent to the /api/chat/{id}/read endpoint and broadcasted as MessageRead.

Next Steps: Store Deployment

Once you are ready to publish, follow these steps:

Google Play Store

  1. Create a Developer Account.
  2. Configure App Icons & Splashes.
  3. Upload the app-release.aab.
  4. Submit for review.

Apple App Store

  1. Join Apple Developer Program.
  2. Setup App ID & Certificates in Xcode.
  3. Upload via Transporter or Xcode.
  4. Submit for Review.
